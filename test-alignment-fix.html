Text Alignment Fix - Verification

What was fixed:

The CustomTextAlign extension was incorrectly calling the parent command, which broke alignment for all paragraphs. The fix changes the command to use the standard TipTap pattern.

Before Fix

// Incorrect - tries to call parent command twice return parentCommands.setTextAlign(alignment)({ state, dispatch });

❌ This pattern doesn't work because parentCommands.setTextAlign is undefined in the extended context

After Fix

// Correct - uses TipTap's command API return commands.updateAttributes('paragraph', { textAlign: alignment });

✅ This uses the standard TipTap command pattern to update paragraph attributes

Expected Behavior

  • Outside blockquotes: Text alignment (left, center, right, justify) should work normally
  • Inside blockquotes: Text alignment should be blocked (buttons disabled or no effect)
  • Creating blockquotes with > : Should use CustomBlockquote with alignment restrictions
  • Creating blockquotes with button: Should use CustomBlockquote with alignment restrictions

Testing Steps

  1. Refresh the editor page to load the updated bundle
  2. Type some text and try to align it (should work)
  3. Create a blockquote using > and try to align (should be blocked)
  4. Create a blockquote using the button and try to align (should be blocked)
  5. Drag aligned text into a blockquote (alignment should be removed)

Technical Details

The fix maintains the blockquote checking logic but properly delegates to TipTap's command system:

  • Checks if cursor is inside a blockquote
  • If yes: returns false to block the command
  • If no: uses commands.updateAttributes('paragraph', { textAlign: alignment })
  • This is the standard pattern for TipTap v3 extensions
